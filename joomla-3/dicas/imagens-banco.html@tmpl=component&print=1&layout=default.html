Evite armazenar imagens no banco de dados


O Joomla nos oferece muitas facilidades. Para inserir uma imagem em um artigo podemos enviar o arquivo da imagem para o site e fazer referência ao mesmo no artigo. Desta forma a imagem ficará guardada junto aos arquivos do site e apenas seu caminho vai para o banco de dados.

Também podemos ter um documento do Word ou Libre Office Writer contendo texto e imagens. Basta copiar todo o documento e colar no artigo do Joomla que todo o conteúdo irá para o artigo, texto e imagens. Desta forma tanto o exto quanto as imagens irão para o banco de dados. Sendo um pequeno artigo, com poucas imagens pequenas não haverá problema, mas se criarmos vários artigos, cada um contendo várias imagens desta forma, que irão para o banco de dados, então poderemos ter problemas.

Tive problemas com este site, por enviar vários artigos assim. O banco cresce muito e alguns artigos chegam a nem abrir no site. O navegador fica tentanto, tentando e não abre.

Também cheguei a receber uma mensagem de erro em um que tinha muitas imagens:

limit reached (pcre.backtrack_limit)

Bastou remover as imagens do banco de dados e inserir como arquivos que tudo foi resolvido.
