Aumentando o tamanho dos uploads no php.ini


Caso use um site ou aplicativo em PHP e tente fazer upload de algum arquivo e receba erro e tenha como alterar o php.ini no servidor, então veja como fazer.

Recebi o erro abaixo ao tentar instalar o componente PhocaGallery no Joomla.

"Erro
Houve um erro ao carregar este arquivo para o servidor.
Tamanho máximo de envio de arquivo PHP é muito pequeno: isso é definido no php.ini nas configurações upload_max_filesize e post_max_size suas do PHP (localizado nos arquivos php.ini e/ou .htaccess)
Não foi possível encontrar o pacote de instalação"

Alguns servidores de hospedagem oferecem a possibilidade de alterar o tamanho dos arquivos no php.ini, outros requerem solicitação de suporte.
Alguns oferecem a possibilidade de se criar um php.ini no raiz do site ou em qualquer dos diretórios.

Se você tiver acesso ao servidor poderá efetuar como descrito abaixo:

Abrir o php.ini, que no Linux Ubuntu com php 7 fica em:

sudo nano /etc/php/7.0/apache2/php.ini

Ou seguindo as regras do seu servidor

Procure por upload_max_filesize (F6 no nano) e altere o valor existente para 8, assim:
upload_max_filesize = 8M

Se 8 não for suficiente aumente mais. Cuidado para não exagerar pois isso compromete a segurança do servidor.
Se for em servidor local não requer muito cuidado, mas é melhor aumentar somente para o necessário.

Aumentar também o post_max_size. Este mantive com 8M.
post_max_size = 8M

Restartar o apache
sudo service apache2 restart

Com isso foi suficiente para instalar o PhocaGallery.
